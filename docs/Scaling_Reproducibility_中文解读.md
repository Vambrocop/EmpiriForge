# 《Scaling Reproducibility》中文解读

> 原文：Yiqing Xu 与 Leo Yang Yang, *Scaling Reproducibility: An AI-Assisted Workflow for Large-Scale Replication and Reanalysis*, 2026。  
> 本解读面向“如何把论文思想转化为 AI skills / 研究工作流 / GitHub 项目”的用途。

## 1. 这篇文章到底在讲什么

这篇文章的核心不是“让 AI 自动做科研判断”，而是把科研复现中大量可程序化、可审计、可重复的环节拆成一组工作流，让大型语言模型负责协调，让确定性代码负责计算，让人类保留科学判断和规则更新的权力。

文章研究的是政治学顶级期刊论文的计算可复现性。作者把 2010-2025 年 APSR、AJPS、JOP 三本期刊中的 3464 篇实证定量论文作为总体，从中抽取 384 篇论文做整篇论文复现，并额外对 92 篇工具变量论文做标准化诊断。

它最重要的结论是：在复现包和可用数据存在时，论文代码本身通常是能跑通、能复现结果的。真正阻碍复现的主要不是作者代码质量，而是复现包缺失、数据受限、链接失效、依赖环境不一致等基础设施问题。

## 2. 文章的三个层次

第一层是制度层。作者讨论 DA-RT、JETS、Dataverse、OSF、ICPSR、期刊第三方验证等制度和基础设施如何提升可复现性。结论很清楚：只要求作者“说有数据”不够，真正有效的是“存档 + 验证”。

第二层是工作流层。文章提出一个 AI-assisted workflow，从论文 PDF 中抽取元数据和数据链接，下载复现包，修复路径和依赖，运行 Stata/R/Python 代码，抽取回归表系数，匹配论文结果和代码结果，再生成诊断报告。

第三层是 skill/agent 层。每个任务被拆成专门角色：Profiler、Librarian、Janitor、Runner、Extractor、Matcher、Classifier、Skeptic、Journalist。每个角色都有输入、输出、工具、约束、失败模式和修复规则。

## 3. 最值得记住的一句话

AI 在这里不是替代研究者，而是把“研究者知道该怎么做，但重复做很贵”的事情工程化。

这就是这篇文章的价值。它不是把 AI 当成“聪明聊天工具”，而是把 AI 放进一个可审计的生产系统：模型负责路由、解释日志和调用工具；确定性程序负责数值计算；人类负责定义研究目标、诊断标准和新增规则。

## 4. 三层架构的含义

文章的系统有三层：

1. **LLM Orchestrator**：大型语言模型编排器，负责读说明、分派任务、看日志、识别错误、选择修复路径。
2. **Skill descriptions and knowledge bases**：每个 agent 的 `SKILL.md`，定义输入输出合同、工具边界、执行约束和历史失败模式。
3. **Deterministic agent code and scripts**：真正执行下载、清洗、运行、估计、诊断、报告生成的 Python/R/Stata 代码。

这个设计最关键的是边界感。LLM 不能直接改数值结果，不能凭感觉估计回归，不能替代统计诊断。它只负责组织和恢复流程。

这对我们设计 skills 很重要：一个好 skill 不应该只是“请你帮我写论文”的提示词，而应该清楚定义：

- 什么时候触发；
- 输入是什么；
- 输出是什么；
- 哪些步骤必须验证；
- 哪些事情不能由模型凭空判断；
- 出错时怎么分类；
- 哪些失败模式可以自动修复，哪些必须交给人。

## 5. Phase A/B/C 工作流

文章把复现分成三个阶段。

**Phase A：获取与执行**

系统从论文 PDF 抽取标题、作者、年份、数据链接，优先使用论文中真实出现的 URL，而不是盲目搜索。然后从 Dataverse、GitHub、OSF 或期刊网站下载复现包，修复路径、依赖、图形输出、旧语法和交互命令，再执行完整代码库。

**Phase B：可复现性验证**

系统从论文回归表中抽取所有点估计，把代码运行得到的系数与论文报告的系数比较。匹配时按照论文报告的小数位数进行精度感知舍入，再做一对一最优匹配。最后给出论文级 verdict：fully、largely、partially、not reproducible。

**Phase C：设计诊断**

如果论文复现成功，并且研究设计属于支持范围，就进入诊断阶段。文章展示的是 IV 诊断，包括第一阶段 F 统计量、Anderson-Rubin 检验、bootstrap 置信区间、jackknife 敏感性、OLS 与 2SLS 对比。

## 6. 主要数字

文章抽样复现了 384 篇论文。

- 275 篇有公开复现文件。
- 251 篇有可访问数据和可执行代码。
- 在这 251 篇中，237 篇完全可复现，比例为 94.4%。
- 384 篇总体中，109 篇没有复现包，24 篇被受限或专有数据阻断。
- 完整论文可复现率从 DA-RT 前的 29.6% 提升到 DA-RT 后的 79.8%。

这组数字的含义是：复现失败常常不是因为论文作者的回归代码不可救，而是因为复现材料没有被制度化保存和验证。

## 7. 文章最强的启发

第一，科研复现可以变成基础设施。过去复现依赖个别研究者花几年手动整理数据、跑代码、核对结果；现在可以把这个过程标准化、日志化、版本化。

第二，skill 是一种组织知识的方式。文章中的 `SKILL.md` 不是简单提示词，而是“可执行工作制度”：它记录输入输出合同、工具边界、失败模式和修复策略。

第三，失败模式比成功案例更宝贵。路径错误、旧版 Stata 语法、缺依赖、变量名不一致、数据格式转换、图形命令阻塞、R 包退役，这些都是下一次自动化的燃料。

第四，AI workflow 的价值在于降低边际成本。手工复现一个项目很贵，但一旦失败模式被编码，下一篇论文的成本就下降。

第五，验证和解释要分开。系统可以告诉你“代码复现了论文表格”，但不能自动证明识别假设成立。尤其是 IV、DID、RDD 这类设计，工具能做诊断，不能替代经济学判断。

## 8. 对经济学写作的启示

这篇文章完全可以转化为一个经济学论文 workflow 范式。经济学论文从想法到投稿，本质上也有一系列可标准化环节：

1. 研究问题是否清楚；
2. 文献位置是否明确；
3. 识别策略是否可信；
4. 数据构造是否可审计；
5. 回归表是否和代码一致；
6. 稳健性检验是否覆盖核心威胁；
7. 图表是否支持主要论点；
8. 写作是否围绕一个中心贡献；
9. replication package 是否能一键运行；
10. 投稿前是否经过审稿人式审查。

因此，你可以做的不是“又一个经济学写作 prompt”，而是一个面向实证经济学的 AI research operating system。

## 9. 可以形成的项目定位

建议定位为：

**EmpiriForge｜实证研究熔炉：面向实证经济学与社会科学的可复现 AI Skills 工作流。**

英文 tagline 可以是：

**Agent Skills for empirical research, causal inference, and reproducible economics writing.**

中文 tagline 可以是：

**把研究经验、识别诊断、复现规范和写作流程锻造成可复用的 AI Skills。**

这个定位有意义，因为现有项目多半有三个缺口：

- 很多项目偏英文，对中文研究者不友好；
- 很多项目偏写作建议，不覆盖数据、回归、复现包；
- 很多项目偏单一 skill，不强调从论文到代码到报告的端到端工作流。

## 10. 写“启示”时可以用的角度

你以后写文章或笔记，可以从这几个角度展开：

**从 prompt 到 workflow**  
AI 的研究价值不在于一次性回答，而在于把重复的专业流程沉淀成可运行、可验证、可改进的工作流。

**从经验到 skill**  
资深研究者脑子里的隐性经验可以被拆成触发条件、输入输出、检查清单、失败模式和修复规则。

**从复现到再分析**  
复现不是终点。复现包一旦被标准化，就可以支持大规模再分析、方法比较和新诊断。

**从个人努力到制度基础设施**  
高可复现率不是靠作者自觉，而是靠期刊要求、数据仓库、第三方验证和自动化工具共同实现。

**从 AI 自动化到人类审计**  
AI 可以降低执行成本，但科学判断仍然必须由人类或明确的学术标准约束。

## 11. 局限和警惕

这篇文章并不意味着“AI 能判断论文是否可信”。它只能判断在给定代码和数据下，结果是否能被重新执行，以及某些标准诊断是否通过。

它也没有解决受限数据、商业数据、政府许可数据、超大计算量模拟、贝叶斯 MCMC 等问题。这些仍需要数据协议、远程安全执行、合成数据或更高计算资源。

另外，自动匹配点估计并不等于完整复现全部论文内容。标准误、图形、描述性统计、附录结果、文本解释和识别假设仍需要扩展。

## 12. 我的总结

这篇文章对你最有价值的地方，是提供了一个构建 skills 的思想模板：

不是把 AI 当成“会说话的研究助理”，而是把它放在一个由 skills、脚本、日志、诊断、报告和版本控制组成的系统里。

你要做的 EmpiriForge 可以从这里出发：先服务经济学实证论文，把研究流程拆成可安装、可复用、可审计的 skills，再逐步扩展到环境、生态、生命科学、组学和 meta 分析。
